# OpsPilot (运维副驾驶) 全量产品需求文档 (PRD)

## 1. 产品定义
* **定位**：自动化部署方案生成工具。
* **核心功能**：根据 `上线checklist.xlsx` 解析结果，自动生成与 `实施文档.docx` 样例**结构和样式完全一致**的 Word 实施文档。
* **核心理念**：人机协同（Copilot），现阶段遵循“机器预判、人工确认、最终生成”的模式。
* **样板基准**：`docs/Sample_Files/实施文档.docx` 为输出的黄金参考，生成文档须在章节结构、表格布局、标题层级、样式上与之对齐。

---

## 2. 输入数据规范 (Input)
* **数据源**：`上线checklist.xlsx`（多 Sheet 结构的 Excel 文件）。
* **Sheet 1（固定）**：**上线安排**，作为整体上线计划表，需以表格形式输出至 Word 第2章「实施步骤和计划」。
* **其余 Sheet**：上线代码包、应用配置、MQS配置、容器配置、集成任务配置等，用于生成详细步骤内容。
* **解析字段**：必须提取 `任务名`、`操作类型`、`部署单元`、`执行/复核人`、`外部链接` 等核心列。
* **外部配置 (`rules.yaml`)**：
    * [cite_start]**优先级 (Priority)**：定义各 Sheet 在 Word 章节中的先后顺序 [cite: 1]。
    * [cite_start]**动作映射 (Mapping)**：定义“操作类型”关键字（如：新增、删除）对应的详细步骤描述文本 [cite: 1]。

## 3. 核心功能需求 (Functional Requirements)

### 3.1 章节排序与过滤
* [cite_start]**自动排序**：系统需读取配置，按权重排列 Word 章节顺序（如数据库脚本优先于应用配置） [cite: 1]。
* [cite_start]**空数据处理**：若 Excel 中某个 Sheet 为空，则自动跳过该章节渲染，不生成空白标题 [cite: 1]。

### 3.2 任务聚合与表格渲染 (核心算法)
* **聚合原则**：同一章节下，相同“操作类型”的任务必须合并处理。
* **展示结构**：
    1. **操作前缀**：展示该操作对应的标准步骤说明（来自动作库映射）。
    2. **任务清单表格**：紧随其后生成表格，列出涉及该操作的所有任务行。
* **动态兼容**：表格表头需根据 Sheet 类型动态对齐（如 MQS 配置展示 Topic，应用配置展示键值对）。

### 3.3 风险识别与协同
* [cite_start]**高危预警**：自动扫描并拦截 `删除`、`下线`、`重建` 等动作关键字 [cite: 1]。
* [cite_start]**链接嗅探**：提取所有超链接（URL），供用户进行时效性检查 [cite: 1]。

### 3.4 Web 配置中心

#### 3.4.1 产品定位
为运维人员提供可视化的规则配置界面，替代直接编辑 `config/rules.yaml`，降低配置出错率。

#### 3.4.2 启动方式
* 新增 CLI 命令：`python main.py web`
* 启动后自动打开浏览器访问本地 Web UI（默认 `http://localhost:8080`）
* 适用场景：单人使用，无需并发控制

#### 3.4.3 功能模块

##### A. 章节排序配置
* **展示**：以列表形式展示已配置的 Sheet 及其优先级
* **多选与批量删除**：
    * 每行左侧增加复选框，支持多选
    * 底部工具栏显示「批量删除」按钮
    * 批量删除需二次确认
* **新增**：手动输入 Sheet 名称，**必须手动指定优先级数值**
* **排序**：支持拖拽调整顺序，自动重新计算优先级
* **删除**：支持删除已配置的章节

##### B. 操作类型配置
* **章节绑定机制**：操作类型必须与章节（Sheet）绑定，每个章节拥有独立的操作类型配置
    * 例如：「应用配置」章节可配置「新增」「修改」「删除」三种操作
    * 例如：「MQS配置」章节可配置「新增订阅」「取消订阅」「修改参数」等操作
* **展示结构**：
    * 左侧：章节列表（可切换）
    * 右侧：当前章节下的操作类型表格
* **多选与批量删除**：
    * 操作类型表格每行左侧增加复选框，支持多选
    * 底部工具栏显示「批量删除」按钮
    * 批量删除需二次确认
* **编辑字段**：
    * 操作类型名称（必填，同一章节内唯一）
    * 步骤说明（支持富文本，**可插入图片**）
        * 富文本编辑器使用 **WangEditor**（MIT 协议，完全免费）
        * 支持基础格式化（加粗、斜体、列表等）
        * 支持图片上传（保存至 `config/images/` 目录）
    * 是否高危（开关）
    * 是否渲染任务清单表格（开关，默认开启）
* **高危操作配置弹窗**：
    * 当标记为高危时，弹出独立配置窗口
    * 支持填写高危操作注意事项（富文本 + 图片）
    * 自动同步到 `high_risk_keywords` 列表
* **图片存储**：插入的图片保存至 `config/images/` 目录
* **配置存储结构**（`rules.yaml` 示例）：
    ```yaml
    action_library:
      应用配置:        # 章节名作为 Key
        新增:          # 操作类型
          instruction: "登录配置中心，进入应用配置页面..."
          is_high_risk: false
          render_table: true
        删除:
          instruction: "确认影响范围后，执行删除操作..."
          is_high_risk: true
          render_table: true
      容器配置:
        扩容:
          instruction: "进入容器管理平台..."
          is_high_risk: false
          render_table: true
    ```

##### C. 列映射配置
* **Sheet 选择**：下拉选择目标 Sheet 类型
* **列配置**：
    * 展示该 Sheet 的输出列配置
    * 每列支持配置多个别名（标签形式）
* **多选与批量删除**：
    * Sheet 列表每行左侧增加复选框，支持多选
    * 底部工具栏显示「批量删除」按钮
    * 批量删除需二次确认
* **别名来源**：
    * 手动输入（主要方式）
    * 可选：上传 Excel 文件辅助识别列名，勾选映射
* **Excel 辅助识别 - 一键保存**：
    * 上传 Excel 后展示识别到的所有 Sheet 及其列名
    * **列名删除**：用户可手动删除不需要在 Word 文档中体现的列
    * **保存到配置按钮**：一键将识别结果保存到配置文件
    * **保存模式**：**全量覆盖**（基于 Excel 内容重新生成配置，清理不存在项）
    * **Sheet 列映射**：全量覆盖 `sheet_column_mapping`，删除 Excel 中不存在的 Sheet 配置
    * **章节排序**：全量覆盖 `priority_rules`（跳过第一个 Sheet），删除 Excel 中不存在的章节配置
    * **优先级自动计算**：按 Sheet 顺序自动分配优先级（10, 20, 30...）
    * **操作类型配置**：保留现有 `action_library`，不删除任何操作类型配置
    * **核心字段**：全量重新生成 `core_fields`，根据保留的列名重新生成，删除不存在项
* **核心字段同步**（自动同步）：
    * **同步时机**：保存列映射配置时**自动执行同步**，无需手动触发
    * **同步规则**：
        * 遍历所有 Sheet 的列名（`sheet_column_mapping` 中已配置的列），收集去重后的列名列表
        * 更新 `core_fields` 中预置字段的 `aliases`：
            * `action_type.aliases`：包含"操作类型"、"操作"、"Action"等关键词的列名
            * `deploy_unit.aliases`：包含"部署单元"、"应用名"、"服务名"等关键词的列名
            * `executor.aliases`：包含"执行人"、"实施人"、"负责人"等关键词的列名
            * `task_name.aliases`：包含"任务名"、"任务名称"等关键词的列名
        * 预置字段的 `required` 属性保持不变
    * **交互反馈**：保存成功后显示 Toast 提示"配置已保存，已同步 N 个核心字段"
    * **设计目的**：确保解析器只识别用户实际配置的列名，避免因列名已被删除但 core_fields 仍包含旧别名导致解析异常

##### D. 其他配置
* 实施总表配置（implementation_summary）
* 输出样式配置（output_config）
* 渲染策略配置（render_config）

#### 3.4.4 配置保存与回滚
* **即时生效**：保存时直接写入 `config/rules.yaml`，立即生效
* **版本历史**：
    * 每次保存自动备份，保留最近 **10 个版本**
    * 备份文件命名格式：`rules.yaml.bak.{timestamp}`
    * 存储位置：`config/backups/`
* **回滚**：支持查看历史版本并一键回滚

#### 3.4.5 交互规范
* 保存前进行表单校验（必填项、格式校验）
* 关键操作（删除、回滚）需二次确认
* 保存成功后显示 Toast 提示
* 支持配置预览（YAML 格式预览）

## 4. 交互需求 (Interaction)
* **两阶段执行**：程序需分为“分析（Analyze）”与“生成（Generate）”两个独立阶段。
* [cite_start]**挂起确认**：在生成 Word 前，Agent 需输出分析摘要（含任务统计、风险提示、链接列表），必须获得人工确认指令后方可触发渲染 [cite: 1]。

---

## 5. 文档输出规范 (Output)
* **格式要求**：标准 `.docx` 格式。
* **内容结构**（须与 `实施文档.docx` 样例一致）：
    * **第1部分**：原因和目的（应用信息、变更原因）。
    * **第2部分**：实施步骤和计划
        * **实施总表**：将 Excel **Sheet 1（上线安排）** 解析为表格，作为本章节的主体内容。
        * **详细步骤**：根据其余 Sheet 的 Excel 解析结果，按操作类型聚合后生成「操作说明 + 任务清单表格」。
    * **第3-5部分**：验证计划、回退措施、风险分析（预留标准模版内容）。
* **样式要求**：
    * 标题层级、表头加粗、表格边框等须与样例文档一致。
    * 保持超链接的可点击活性。

## 6. 验收标准 (Acceptance Criteria)

### 6.1 核心功能验收
* **与样例对齐**：生成的 Word 文档在章节结构、表格形式、排版样式上须与 `实施文档.docx` 可视觉比对一致。
* **实施总表来源**：第2章「实施步骤和计划」中的实施总表必须来自 Excel Sheet 1（上线安排），不得缺失或错位。
* **逻辑闭环**：详细步骤中的任务明细必须与 Excel 源数据及配置严格对应。
* **聚合正确**：同类操作必须成功聚合，不得出现操作说明重复输出的情况。
* **安全拦截**：所有定义的高危操作必须在对话环节被准确识别并提示确认。

### 6.2 Web 配置中心验收

#### 启动与访问
* 执行 `python main.py web` 后，服务在 5 秒内启动完成
* 浏览器自动打开 `http://localhost:8080`，显示配置中心首页

#### 章节排序配置
* 新增章节时，未填写优先级无法保存（必填校验）
* 拖拽排序后，优先级数值自动重新计算
* 保存后 `rules.yaml` 中 `priority_rules` 节点正确更新
* **批量删除**：多选章节后点击批量删除，二次确认后正确移除

#### 操作类型配置
* **章节绑定**：操作类型必须与章节关联，切换章节后显示该章节的操作类型列表
* **独立配置**：不同章节可配置同名操作类型（如「新增」），但步骤说明独立
* **批量删除**：多选操作类型后点击批量删除，二次确认后正确移除
* 富文本编辑器（WangEditor）支持插入图片，图片保存至 `config/images/`
* 标记高危后，高危配置弹窗正常弹出
* 高危操作自动同步到 `high_risk_keywords` 列表
* 图片在步骤说明中正确显示
* 保存后 `rules.yaml` 中 `action_library` 结构为 `{章节名: {操作类型: 配置}}`

#### 列映射配置
* 手动输入别名可正常添加和删除
* **批量删除**：多选 Sheet 后点击批量删除，二次确认后正确移除
* 上传 Excel 后，列名识别准确，勾选映射后正确配置
* 保存后 `sheet_column_mapping` 节点结构正确
* **Excel 一键保存（全量覆盖模式）**：
    * 点击"保存到配置"后，基于 Excel 内容全量覆盖 `sheet_column_mapping` 和 `priority_rules`
    * Excel 中不存在的 Sheet 配置被自动清理
    * Excel 中不存在的章节从 `priority_rules` 中删除（跳过第一个 Sheet）
    * `action_library` 保持不变，不删除任何操作类型配置
    * `core_fields` 基于新列名全量重新生成，旧的自定义字段被清理
    * 优先级按 Sheet 顺序自动分配（10, 20, 30...）
    * 保存成功后显示 Toast 提示
* **核心字段同步**（自动同步）：
    * 保存列映射配置时自动执行同步，无需手动触发
    * 同步后，`action_type.aliases` 包含含"操作类型"关键词的列名
    * 同步后，`task_name.aliases` 包含含"任务名"关键词的列名
    * 预置字段的 `required` 属性保持不变
    * 保存成功后显示 Toast 提示"配置已保存，已同步 N 个核心字段"
    * 解析器使用同步后的 core_fields 别名能正确匹配 Excel 列名

#### 配置保存与回滚
* 保存时自动生成备份文件至 `config/backups/`
* 备份文件命名格式正确：`rules.yaml.bak.{YYYYMMDD_HHMMSS}`
* 历史版本列表最多显示 10 条，超出时自动删除最旧版本
* 回滚操作可正确恢复指定版本的配置
* 回滚需二次确认，确认后立即生效

#### 通用交互
* 所有必填项为空时，保存按钮禁用或提示错误
* 删除操作需二次确认弹窗
* 保存成功后显示 Toast 提示「配置已保存」
* YAML 预览功能正常，格式正确

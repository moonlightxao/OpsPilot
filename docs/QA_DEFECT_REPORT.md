## 缺陷清单

### 缺陷 #1 【P0】实施总表列异常

| 项目 | 内容 |
|------|------|
| **标准** | 列应为：序号 \| 任务 \| 开始时间 \| 结束时间 \| 实施人 \| 复核人 |
| **实际** | 出现 `46315`、`Unnamed: 6`～`Unnamed: 14`、`46316` 等列 |
| **根因** | `report.json` 中 `implementation_summary.columns` 未正确映射与过滤 |

**复现**：
```
output/report.json 中 implementation_summary.columns 含：
["任务序号", "任务", "是否完成", "实施人", "复核人", "46315", "Unnamed: 6", ...]
```

**修正建议**：
- Parser：`drop_unnamed_columns` 未对黄金样本 Excel 生效，需排查 `上线安排` / `变更安排` Sheet 的列解析
- Parser：列 `46315`、`46316` 疑似 Excel 日期序列号被误解析为列名，需过滤或转换
- rules.yaml：`column_mapping` 需覆盖黄金样本中的 `任务序号`→`序号`、补充日期列映射

---

### 缺陷 #2 【P0】第2部分层级与命名不符

| 项目 | 内容 |
|------|------|
| **标准** | 2 实施步骤和计划 → 2.1 详细实施步骤 → 2.1.1～2.1.N 各子章节 |
| **实际** | 2.1 实施总表、2.2 详细步骤 |
| **根因** | 模板 `template.docx` 结构未按 TECH_DESIGN v2 修正 |

**复现**：
```
生成文档 headings: ['2.1 实施总表', '2.2 详细步骤']
标准应为: ['2 实施步骤和计划', '2.1 详细实施步骤', '2.1.1 xxx', '2.1.2 xxx', ...]
```

**修正建议**：
- `scripts/create_template.py`：将「2.1 实施总表」改为在「2 实施步骤和计划」下的无编号表格区块
- 将「2.2 详细步骤」改为「2.1 详细实施步骤」，子节使用 `2.1.{{ loop.index }}` 编号

## 新发现缺陷（2026-02-24）

### 缺陷 #4 【P0】第 2.1 详细实施步骤内容为空

| 项目 | 内容 |
|------|------|
| **标准** | 根据 PRD 第 5 节与样例文档，第 2.1「详细实施步骤」下，每个子章节应包含「操作说明 + 任务清单表格」，表格行由 Excel 各业务 Sheet 聚合而来 |
| **实际** | 使用黄金样本生成的 `output/实施文档.docx` 中，第 2.1 详细实施步骤区域仅有章节标题/占位，无任何任务表格或明细行，整体为空 |
| **影响** | 用户无法在实施文档中看到按操作类型聚合后的任务清单，核心业务价值缺失，属 **P0 级阻断缺陷** |

**复现**（QA 本地环境）：

```bash
# 1. 生成实施文档
python main.py run docs/Sample_Files/上线checklist.xlsx -o output/实施文档.docx

# 2. 打开 output/实施文档.docx
#    - 定位到「2 实施步骤和计划」→「2.1 详细实施步骤」
#    - 观察各 2.1.x 子章节下是否存在任务表格与明细行
```

**预期行为**：
- 每个业务章节（如「数据库脚本部署」「应用配置」等）下，应存在至少一张任务表格
- 表格列应与 `report_schema.md` 中 section.tasks 约定一致，且行数与 Excel 源数据对应

**当前结论**：
- 问题已由 QA 复现并确认，与样例文档对比后判定不符合 PRD 第 5 节「详细步骤」输出规范
- **根因**：待 Developer 排查（可能涉及 `report.json` 中 sections 填充或 `TemplateRenderer` 对 section.tasks 的渲染逻辑）
